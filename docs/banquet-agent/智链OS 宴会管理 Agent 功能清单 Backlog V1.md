# **《智链OS 宴会管理 Agent 功能清单 Backlog V1》**





版本：V1.0

产品名称：智链OS Banquet Agent

文档类型：研发排期版 Backlog

目标：支持连锁餐饮大、中、小型宴会的全流程管理，并形成宴会经营智能体基础能力



这份 Backlog 不是按“页面”拆，而是按 **Epic → Feature → User Story → 优先级 → 验收标准** 来拆，方便产品、研发、测试、实施统一协作。



------





# **一、V1 范围定义**





V1 聚焦四件事：



1. 宴会线索到成交全流程
2. 宴会档期、厅房、套餐、订单一体化管理
3. 宴会执行协同与任务拆解
4. 宴会收入、成本、利润、转化率的基础分析





V1 暂不追求：



- 复杂婚庆全案系统
- 灯光舞美深度排期
- 跨品牌大型会展管理
- 高级 AI 自动谈单
- 全自动动态定价引擎





------





# **二、Backlog 总体结构**



```
Epic A 宴会客户与线索中心
Epic B 宴会档期与厅房资源中心
Epic C 宴会套餐与菜单中心
Epic D 宴会订单与合同收款中心
Epic E 宴会执行任务中心
Epic F 宴会成本与利润中心
Epic G 宴会经营分析中心
Epic H 宴会 Agent 编排中心
Epic I 角色工作台与驾驶舱
Epic J 系统集成与消息中心
Epic K 权限、审计与基础配置
```

# **Epic A 宴会客户与线索中心**





目标：把宴会业务从“记本式销售”升级成可管理的 CRM 漏斗。





## **Feature A1 客户档案管理**





**User Story**

作为宴会销售，我希望维护客户档案，方便持续跟进和复购营销。



**功能点**



- 新建客户
- 编辑客户信息
- 联系人管理
- 客户标签
- 历史宴会记录查看
- 客户来源记录





**优先级**：P0



**验收标准**



- 可创建个人客户、企业客户
- 可记录至少 3 个联系人字段（姓名、电话、关系/职位）
- 可查看客户历史宴会订单





------





## **Feature A2 宴会线索录入**





**User Story**

作为宴会销售，我希望快速记录来电、微信、抖音、美团等渠道线索。



**功能点**



- 线索来源选择
- 宴会类型
- 人数
- 预算
- 意向日期
- 意向门店
- 厅房偏好
- 跟进状态





**优先级**：P0



**验收标准**



- 线索可从手工录入创建
- 可支持来源分类统计
- 可支持线索状态流转





------





## **Feature A3 跟进记录**





**User Story**

作为宴会销售，我希望记录每次跟进内容，避免客户丢失。



**功能点**



- 电话记录
- 微信沟通记录摘要
- 看厅记录
- 报价发送记录
- 下次跟进时间





**优先级**：P0



**验收标准**



- 客户详情页可看到完整时间线
- 每次跟进可记录摘要、结果、下次动作





------





## **Feature A4 销售漏斗看板**





**User Story**

作为宴会经理，我希望看到线索到成交的漏斗，判断团队转化效率。



**漏斗阶段建议**



- 新线索
- 已联系
- 已看厅
- 已报价
- 待确认
- 已定金
- 已成交
- 已流失





**优先级**：P1



**验收标准**



- 可按门店、销售人员、时间筛选
- 可看到各阶段数量与转化率

# **六、V1 成功标准**







## **业务成功**





- 宴会线索不再靠纸和微信散落管理
- 档期冲突明显下降
- 宴会准备协同效率提升
- 单场宴会利润可见
- 管理层能看到宴会收入和转化







## **系统成功**





- 能完整跑通：线索 → 报价 → 订单 → 执行 → 结算 → 分析
- 多角色可用
- 多门店可配置
- 基础 Agent 可触发、可提醒、可闭环







## **管理成功**





- 销售知道先跟进谁
- 店长知道今天要准备什么
- 财务知道哪些单该收钱
- 老板知道宴会是赚是亏

# **七、Backlog 一句话总结**





**智链OS 宴会管理 Agent Backlog V1** 的本质是：



> 先把宴会业务从“靠经验、靠人盯、靠群协调”

> 变成“有线索、有资源、有任务、有利润、有分析”的数字化经营系统，

> 再逐步升级为 Agent 驱动的宴会经营操作系统。
