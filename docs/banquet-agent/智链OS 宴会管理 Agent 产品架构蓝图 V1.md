# **《智链OS 宴会管理 Agent 产品架构蓝图 V1》**





版本：V1.0

产品名称：智链OS Banquet Agent

产品定位：面向连锁餐饮的大、中、小型宴会经营智能体系统

设计目标：把“宴会预订工具”升级为“宴会经营操作系统”



------





# **一、蓝图总述**





智链OS 宴会管理 Agent，不应该只做：



- 宴会客户登记
- 宴会厅档期管理
- 套餐与报价





这些只是工具层。



真正的目标应该是：



> 把宴会从“销售驱动、人工协调、经验执行”

> 升级为“数据驱动、智能协同、可预测、可复盘”的经营系统。



所以，智链OS宴会管理 Agent 的产品架构，必须覆盖五个层级：

```
智链OS 宴会管理 Agent
│
├── A. 宴会线索与客户层
├── B. 宴会订单与资源层
├── C. 宴会执行协同层
├── D. 宴会经营分析与预测层
└── E. 宴会 Agent 编排层
```



------





# **二、产品北极星目标**





宴会系统最终要回答五个核心问题：



1. 哪些客户最容易成交
2. 哪个厅、哪个时段最适合接什么宴会
3. 这场宴会要准备什么、谁来做、何时完成
4. 这场宴会到底赚不赚钱
5. 本月宴会收入、利润、产能还能提升多少





也就是说，它既要服务一线执行，也要服务老板经营决策。

# **四、五层架构详细拆解**







## **A. 宴会线索与客户层**





这一层的目标不是简单“录客户”，而是建立宴会 CRM。





### **核心对象**





- 客户
- 联系人
- 宴会线索
- 渠道来源
- 意向宴会需求
- 跟进记录
- 成交状态







### **核心能力**







#### **1）客户档案**



记录：



- 姓名 / 企业 / 家庭关系
- 联系方式
- 历史宴会记录
- 偏好
- 预算带
- 宴会类型偏好







#### **2）线索池**



按来源进入：



- 电话
- 微信
- 抖音
- 美团
- 小红书
- 老客转介绍
- 到店咨询







#### **3）需求采集**



关键字段：



- 宴会类型
- 日期
- 人数
- 预算
- 厅房偏好
- 菜系偏好
- 酒水需求
- 布场需求
- 特殊禁忌







#### **4）销售漏斗**



阶段建议：

```
新线索
→ 已联系
→ 已邀约
→ 已看厅
→ 已报价
→ 待确认
→ 已定金
→ 已成交
→ 已流失
```



### **这一层的核心价值**





把宴会业务从“接电话记本子”变成“可管理的成交漏斗”。



------





## **B. 宴会订单与资源管理层**





这一层是传统宴会系统最常见的部分，但在智链OS里必须更深。





### **核心对象**





- 宴会订单
- 宴会厅 / 包间
- 档期
- 套餐
- 菜单
- 定金
- 尾款
- 桌数
- 人数
- 资源占用







### **核心能力**







#### **1）宴会订单中心**



一个宴会订单至少要包含：



- 宴会类型
- 日期与时间段
- 门店
- 厅房
- 桌数 / 人数
- 套餐 / 菜单
- 酒水
- 服务要求
- 定金状态
- 合同状态
- 执行状态







#### **2）宴会档期管理**



支持：



- 宴会厅日历视图
- 包间日历视图
- 门店档期总览
- 冲突提醒
- 可售时段查询





要能回答：



- 哪个厅在某天某时是否空闲
- 同一时间是否已超负荷
- 某厅更适合接婚宴还是商务宴







#### **3）套餐与菜单管理**



分类建议：



- 婚宴套餐
- 商务宴套餐
- 寿宴套餐
- 满月宴套餐
- 团建宴套餐
- 自定义宴会菜单





每个套餐必须关联：



- 菜品
- 成本
- 建议售价
- 毛利率
- 适配人数
- 适配厅型







#### **4）合同与收款管理**



支持：



- 定金收取
- 合同生成
- 节点付款提醒
- 尾款结算
- 开票状态







### **这一层的核心价值**





把“档期、菜单、价格、订单”统一到一个结构里，为后续执行与利润分析打底。



------





## **C. 宴会执行协同层**





这层是餐饮宴会最容易出问题的地方，也是智链OS真正拉开差距的地方。





### **核心对象**





- 宴会执行任务
- 厨房任务
- 服务任务
- 采购任务
- 布场任务
- 设备任务
- 现场异常







### **核心能力**







#### **1）宴会任务拆解**



系统根据订单自动拆出任务：



- 厨房备菜
- 原料采购
- 酒水备货
- 厅房布置
- 音响屏幕检查
- 餐具与桌卡准备
- 服务员排班
- 迎宾安排
- 现场负责人指定







#### **2）时间轴执行管理**



按时间轴组织任务：

```
T-7天：菜单确认 / 采购初单
T-3天：厅房确认 / 人员排班
T-1天：布场 / 备菜 / 酒水准备
T-3小时：现场检查 / 人员到位
执行中：上菜 / 服务 / 异常响应
结束后：结算 / 清场 / 评价
```



#### **3）多部门协同**



需要与以下系统联动：



- 采购系统
- 库存系统
- 人力排班系统
- 财务收款系统
- 门店运营系统







#### **4）异常处理**



异常类型：



- 厅房冲突
- 菜品缺货
- 客户临时加桌
- 服务人员不足
- 酒水不足
- 上菜节奏延误
- 设备故障





系统要支持：



- 异常上报
- 指派责任人
- 响应时间记录
- 处理闭环







### **这一层的核心价值**





把宴会从“靠群里喊人”变成“有流程、有任务、有节奏”的执行系统。



------





## **D. 宴会经营分析与预测层**





这层决定宴会系统是不是“经营系统”。





### **核心对象**





- 宴会收入
- 宴会成本
- 宴会利润
- 档期利用率
- 客户转化率
- 客户来源
- 热门套餐
- 复购率







### **核心能力**







#### **1）收入分析**



维度包括：



- 按门店
- 按宴会类型
- 按厅房
- 按时间
- 按渠道来源
- 按销售人员







#### **2）利润分析**



利润模型建议：

```
宴会利润 = 宴会收入
         - 食材成本
         - 酒水成本
         - 人工成本
         - 布场及赠送成本
         - 其他直接成本
```



#### **3）转化分析**



漏斗分析：

```
咨询数
→ 有效需求数
→ 看厅数
→ 报价数
→ 定金数
→ 成交数
```



#### **4）档期产能分析**



分析：



- 高峰档期饱和度
- 空档浪费
- 热门厅利用率
- 小厅/中厅/大厅资源效率







#### **5）收入预测**



预测本月：



- 宴会收入
- 成交率
- 空余产能
- 潜在增量







#### **6）客户复购与推荐分析**



识别：



- 老客再次办宴会概率
- 高价值客户群
- 转介绍来源质量







### **这一层的核心价值**





让老板不只是知道“接了多少单”，而是知道“哪里能多赚、哪里在浪费”。



------





## **E. Agent 编排与智能决策层**





这是智链OS宴会管理 Agent 的核心差异化。



不是功能模块堆积，而是由 Agent 驱动业务。



------





# **五、宴会 Agent 体系设计**





建议 V1 建立 12 个核心 Agent。





## **1）宴会客户 Agent**





负责：



- 线索评分
- 客户分层
- 成交概率判断
- 老客识别
- 跟进提醒







## **2）宴会需求识别 Agent**





负责把客户描述转成结构化需求：



- 宴会类型
- 人数
- 预算
- 日期
- 菜单偏好







## **3）宴会报价 Agent**





根据：



- 人数

- 厅房

- 菜单

- 酒水

- 时段

  自动生成报价建议。







## **4）宴会排期 Agent**





负责：



- 空闲厅房匹配
- 时间冲突校验
- 不同宴会类型与厅型的最佳分配







## **5）宴会菜单 Agent**





根据宴会类型、预算、人数推荐套餐与菜单组合。





## **6）宴会资源 Agent**





负责计算需要的：



- 服务员数量
- 厨师负荷
- 餐具
- 酒水
- 布场物料
- 厨房准备能力







## **7）宴会执行 Agent**





根据订单生成执行清单和时间轴任务，并推动协同。





## **8）宴会成本 Agent**





自动计算：



- 食材成本
- 人工成本
- 赠送成本
- 真实毛利







## **9）宴会收入预测 Agent**





按线索池、档期和历史转化预测未来收入。





## **10）宴会营销 Agent**





分析：



- 哪个渠道来客最多
- 哪个渠道成交最好
- 哪类宴会增长最快







## **11）宴会体验 Agent**





收集：



- 评价

- 投诉

- 延迟

- 客诉

  并形成体验评分。







## **12）宴会复盘 Agent**





自动输出：



- 成本复盘
- 执行复盘
- 客诉复盘
- 利润复盘
- 可优化项

# **七、宴会知识图谱（Banquet Ontology）**





如果要走“餐饮 Palantir”路线，宴会系统不能只有表单，必须有语义模型。





## **核心节点**





- Customer
- Lead
- BanquetOrder
- BanquetType
- BanquetHall
- TimeSlot
- MenuPackage
- Dish
- Ingredient
- Staff
- Task
- Payment
- Contract
- Feedback
- Complaint
- Channel







## **核心关系**



```
Customer → HAS_LEAD → Lead
Lead → CONVERTS_TO → BanquetOrder
BanquetOrder → USES → BanquetHall
BanquetOrder → OCCURS_AT → TimeSlot
BanquetOrder → SELECTS → MenuPackage
MenuPackage → CONTAINS → Dish
BanquetOrder → REQUIRES → Staff
BanquetOrder → GENERATES → Task
BanquetOrder → HAS_PAYMENT → Payment
BanquetOrder → RECEIVES → Feedback
```



## **图谱层价值**





它要支持回答：



- 某类型宴会最常用哪些厅
- 哪类套餐最赚钱
- 哪类客户最容易复购
- 哪些执行异常最容易影响评价
- 哪些厅的高峰时段被低价值宴会占用

## **业务成功**





- 宴会转化率提升
- 档期冲突下降
- 宴会准备效率提升
- 宴会毛利可视化
- 客户复购率提升







## **管理成功**





- 老板能看到本月宴会收入与利润预测
- 店长能知道每场宴会该准备什么
- 销售能知道哪些客户更值得跟进
- 财务能清楚宴会收款与盈利情况





------





# **十二、蓝图一句话总结**





**智链OS 宴会管理 Agent 产品架构蓝图 V1** 的本质是：



> 把宴会从“客户预订与场地安排”

> 升级成“客户、资源、执行、利润、预测”一体化的宴会经营操作系统。



再直白一点：



> 不是帮餐厅“接宴会”，

> 而是帮餐厅“把宴会做成一门高效率、高利润、可复制的生意”。
