# **《屯象OS 宴会管理 Agent UI 信息架构 V1》**





版本：V1.0

产品名称：屯象OS Banquet Agent

文档类型：UI 信息架构 / IA 设计稿

适用对象：产品经理 / UX / UI / 前端 / 研发 / 实施

设计目标：为连锁餐饮宴会业务提供一套可落地、可扩展、可角色分层的 UI 信息架构，支撑从线索到复盘的全流程协同与经营分析



------





# **一、设计目标**





这份 UI 信息架构，不是给“单店宴会预订页面”用的。

它要解决的是：



1. 多门店、多厅房、多销售、多角色协作
2. 宴会业务从线索、报价、订单、执行到结算的完整流转
3. 老板、宴会经理、销售、店长、财务、厨房负责人看到不同的“宴会世界”
4. 为后续 Agent 驱动、预测分析、任务协同、经营复盘预留清晰入口





所以 UI 设计必须遵循四个原则：



- **角色优先**
- **任务优先**
- **状态优先**
- **经营优先**





------

# **四、角色视角架构**





宴会系统不能让所有人看到同一套菜单。

建议按角色提供不同默认入口。



------

```

```

# **六、关键页面之间的交互流**





------





## **6.1 线索到订单交互流**



```
线索池
→ 线索详情
→ 新增跟进 / 看厅记录
→ 生成报价
→ 报价确认
→ 转正式订单
→ 自动锁档
→ 进入订单详情
→ 自动生成执行任务
```



------





## **6.2 档期到建单交互流**



```
宴会日历
→ 点击空闲厅房时段
→ 弹出快速建单侧边栏
→ 选择客户/新建客户
→ 填写人数、宴会类型、预算
→ 选择套餐
→ 生成订单
```



------





## **6.3 订单到执行交互流**



```
订单详情
→ 查看任务时间轴
→ 分派负责人
→ 任务中心执行
→ 异常处理
→ 完成宴会
→ 进入结算 / 评价 / 复盘
```



------





## **6.4 驾驶舱到下钻交互流**



```
宴会驾驶舱
→ 点击收入趋势
→ 收入分析页
→ 点击某门店
→ 门店订单列表
→ 订单详情
```

# **十一、UI 信息架构一句话总结**





**屯象OS 宴会管理 Agent UI 信息架构 V1** 的核心，不是多做几个页面，

而是让不同角色都能在系统里快速完成自己的关键任务：



- 销售看的是成交
- 店长看的是执行
- 财务看的是收款与利润
- 老板看的是收入、产能和增长





再直白一点：



> 这套 IA 不是为了“看起来完整”，

> 而是为了把宴会业务真正跑起来、管起来、分析起来、优化起来。
