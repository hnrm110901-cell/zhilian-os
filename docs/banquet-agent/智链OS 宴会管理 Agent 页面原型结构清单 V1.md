# **《屯象OS 宴会管理 Agent 页面原型结构清单 V1》**





版本：V1.0

类型：UI Wireframe 结构清单

适用：产品经理 / UX / UI / 前端 / 架构 / 实施团队



本文件目标：



> 将宴会管理 Agent 的 UI 信息架构拆成**可直接设计线框图的页面结构**。



每个页面包含：



- 页面目标
- 页面模块结构
- 组件结构
- 关键交互

# **二、01 宴会驾驶舱**







## **页面目标**





老板 / 区域负责人 / 店长快速了解宴会经营情况。



------





## **页面布局结构**



```
┌─────────────────────────────────────────┐
│ 全局筛选栏                               │
│ 门店 | 区域 | 时间范围 | 宴会类型         │
├─────────────────────────────────────────┤
│ KPI指标卡片                              │
│ 收入 | 利润 | 成交数 | 转化率 | 客单价     │
├─────────────────────────────────────────┤
│ 收入趋势图         | 宴会类型收入占比      │
├─────────────────────────────────────────┤
│ 销售漏斗            | 档期利用率            │
├─────────────────────────────────────────┤
│ 高价值订单         | 风险订单               │
├─────────────────────────────────────────┤
│ Agent建议中心                             │
└─────────────────────────────────────────┘
```



------





## **KPI卡片组件**



```
┌─────────────┐
│ 本月宴会收入 │
│ ¥1,200,000  │
│ ↑12%        │
└─────────────┘
```

卡片共 6 个：



- 宴会收入
- 宴会利润
- 成交订单数
- 转化率
- 档期利用率
- 客单价

# **十八、页面结构一句话总结**





宴会管理 Agent UI 的核心逻辑：

```
线索 → 报价 → 锁档 → 订单 → 执行 → 收款 → 复盘
```

UI 的设计必须让用户：



- 一眼知道档期
- 一眼知道订单
- 一眼知道风险
- 一眼知道利润
