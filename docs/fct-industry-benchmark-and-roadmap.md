# 业财税资金一体化行业对标与智链OS完善路线图

本文档基于**行业主流方案**（合思费控、金蝶发票云/星空、业财税四流合一实践等）做能力对标，取长补短，指导智链OS 业财税资金模块的持续完善。

---

## 一、行业方案要点摘要

### 1.1 合思等费控/业财方案（连锁餐饮场景）

| 能力域 | 要点 | 价值 |
|--------|------|------|
| **费用与备用金** | 门店日常费用、固定/临时备用金、易商卡等；申请、冲销、还款全流程 | 零垫资、留痕、降低门店现金风险 |
| **发票管理** | 自动采集、智能识别、验真查重、电子+纸质统一归档；与报销/支付关联 | 合规、减负、票-账-款一致 |
| **预算与项目** | 新店装修/营销活动项目预算；立项、合同、执行、结案；预算编制/冻结/占用/分析 | 事前控制、执行可控 |
| **合同与摊销** | 租金等非生产性合同、20+ 租金计算模型、资产折旧分摊、门店费用分摊后自动凭证 | 自动化、多维度核算 |
| **审批与协同** | 移动端申请、自动审批流、与 OA/ERP/供应链对接；出纳付款后自动生成凭证 | 效率提升、数据同源 |

**典型成效**（行业案例）：审批效率提升约 60%，财务处理成本降低约 40%；多门店场景下总部少量人员即可承接全国核算。

### 1.2 业财税「四流合一」（金税四期语境）

| 流 | 含义 | 关键动作 |
|----|------|----------|
| **业务流** | 订单-发货-收款真实可追溯 | 业务数据实时准确、每笔交易可追溯 |
| **财务流** | 会计凭证自动生成 | 基于业务规则自动生成凭证，与业务一致 |
| **票据流** | 发票与货、款、合同、账务闭环 | 开票与物流、资金、合同、账务实时对应 |
| **税务流** | 税务申报自动取数 | 基于统一数据底座生成申报表与台账 |

**风险**：四流不一致易导致进项无法抵扣、所得税扣除争议、虚开风险及系统孤岛。

**实现路径**：统一数据底座 + 凭证映射规则 + 票-货-款-账自动对账 + 电子发票平台对接 + 税务风险监控。

### 1.3 金蝶等财务/发票方案

- **发票云**：开票、报销、入账全流程数字化；订单数据归集、财务协同。
- **星空餐饮云**：营业收入财务协同、全渠道订单归集、多系统集成。
- **通用**：科目与准则、凭证借贷平衡、辅助核算、多主体多账簿、银企直连等。

## 二、智链OS 当前能力与四流对应

| 四流 | 智链OS 现状 | 说明 |
|------|-------------|------|
| **业务流** | ✅ 事件驱动 | 日结/采购等事件带 `event_id`、`source_id`、`payload`，业务可追溯 |
| **财务流** | ✅ 规则引擎 | 门店日结、采购入库自动生成凭证；总账、按门店/区域/期间汇总 |
| **票据流** | 🔶 部分 | 事件 payload 可传 `invoice_no`、`source_doc_id`、`order_id` 等，生成凭证时写入 `voucher.attachments`，便于票-账-业务单关联；税务发票表占位，待与开票/验真对接 |
| **税务流** | 🔶 部分 | 凭证含销项/进项科目；申报表占位；与税控/全电对接待建 |

| 能力域 | 智链OS 现状 | 行业常见 |
|--------|-------------|----------|
| 凭证与总账 | ✅ 凭证规则、借贷平衡、总账余额、报表汇总/趋势/同比环比 | 同 |
| 资金与对账 | ✅ 资金流水、对账状态；与日结联动 | 银企直连、自动勾对、归集 |
| 主数据 | ✅ 门店/客商/科目/银行账户 | 同 |
| 费控/备用金 | ✅ Phase 4 已落地 | 备用金主档与流水（申请/冲销/还款）、审批记录占位 |
| 预算 | ✅ Phase 4 已落地 | 项目/期间预算、编制与占用接口、超预算校验 |
| 发票闭环 | ✅ Phase 4 已落地 | 发票与凭证关联、验真占位（verify_status）、按凭证查发票 |
| 合并报表 | ❌ 未建 | 多主体合并、内部抵销 |

## 三、取长补短：完善方向与路线图

### 3.1 短期（已做 + 小增强）

- **四流合一追溯**：事件 `payload` 可传 `invoice_no`、`source_doc_id`、`order_id` 等；生成凭证时写入 `voucher.attachments`，便于票-账-业务单关联查询与审计。
- **文档与契约**：业财事件、报表 API、凭证规则与金蝶/用友及准则对照已文档化；本对标与路线图作为规划输入。

### 3.2 中期（Phase 4 规划）— 已落地

以下能力已实现模型、服务与 API（合并形态 `src/api/fct.py`、独立形态 `src/api/fct_public.py` 同契约）。

| 方向 | 内容 | 参考行业 | 接口路径（示例） |
|------|------|----------|------------------|
| **费控与备用金** | 备用金类型（固定/临时）、申请/冲销/还款；与凭证/资金联动；审批流占位 | 合思等 | `PUT/GET /petty-cash`、`POST /petty-cash/records`、`GET /petty-cash/{id}/records` |
| **预算占位** | 项目/期间预算表、预算编制与占用接口；与凭证规则联动（超预算校验/占用） | 合思、金蝶 | `PUT /budgets`、`GET /budgets/check`、`POST /budgets/occupy` |
| **发票闭环** | 发票表与凭证关联、验真占位（verify_status/verified_at）；票-账关联查询 | 金蝶发票云、四流合一 | `POST /invoices/link`、`GET /invoices/by-voucher/{voucher_id}`、`POST /invoices/{id}/verify` |
| **审批与流程** | 审批记录占位（ref_type/ref_id/step/status）；与 OA 或内部工作流对接可扩展 | 合思、OA 集成 | `POST /approvals`、`GET /approvals/by-ref` |

### 3.3 长期（能力扩展）

- **银企直连**：流水自动下载、支付指令、余额查询，与资金与对账模块打通。
- **多主体与合并**：法人账/管理账、主体间往来、合并报表与抵销规则。
- **税务申报取数**：基于总账与发票数据自动生成申报表与台账，对接电子税务局。
