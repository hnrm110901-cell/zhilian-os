# 奥琦玮微生活 · 会员网关接口学习笔记

**官方文档（语雀）**：[会员网关](https://www.yuque.com/acewillomp/mmag8t/iynakxn7fnz3pfkp)  
**用途**：对接尝在一起等品牌在奥琦玮微生活侧的会员、储值、交易、优惠券等能力；与品智收银、奥琦玮供应链区分使用。

---

## 一、文档与系统关系

| 系统 | 文档/地址 | 认证 | 主要能力 |
|------|-----------|------|----------|
| **奥琦玮微生活** | 语雀 · 会员网关（见上链接） | API-AppId + API-AppKey | 会员、储值、交易、优惠券 |
| 奥琦玮供应链开放平台 | openapi.acescm.cn | AppKey + AppSecret | 库存、POS 订单上传、日结 |
| 品智收银 | 品智开放 API（语雀） | Token + 签名 | 门店、经营数据、订单 |

尝在一起使用**奥琦玮微生活**时，认证参数为 **API-AppId**、**API-AppKey**（见 [尝在一起-奥琦玮微生活与门店配置](./尝在一起-奥琦玮微生活与门店配置.md)）。

## 二、会员网关常见能力（供与语雀文档对照）

以下为典型「会员网关」能力，具体以语雀文档为准；可在文档中核对路径、参数与响应格式。

### 2.1 会员管理

| 能力 | 说明 | 本仓库适配器 README 约定 |
|------|------|---------------------------|
| 查询会员 | 按卡号/手机号/openid 查询会员详情 | `query_member(card_no=…)` |
| 新增会员 | 注册新会员（手机、姓名、性别、生日、门店等） | `add_member(mobile, name, sex, birthday, store_id…)` |
| 修改会员 | 更新会员资料 | `update_member(card_no, update_data)` |
| 会员标签/等级 | 标签管理、等级调整 | README 标为 ⏳ |

### 2.2 交易处理

| 能力 | 说明 | 本仓库适配器 README 约定 |
|------|------|---------------------------|
| 交易预览 | 按金额/商品计算优惠、应付 | `trade_preview(card_no, store_id, amount, dish_list…)` |
| 交易提交 | 提交消费（支付方式、抵扣方案） | `trade_submit(card_no, store_id, amount, pay_type, trade_no, discount_plan…)` |
| 交易查询/撤销 | 按单号查询、撤销 | `trade_submit`、交易查询/撤销 README 有述 |

### 2.3 储值管理

| 能力 | 说明 | 本仓库适配器 README 约定 |
|------|------|---------------------------|
| 储值提交 | 充值（金额单位：分） | `recharge_submit(card_no, store_id, amount, pay_type, trade_no)` |
| 储值查询 | 查询余额 | `recharge_query(card_no)` |

### 2.4 优惠券管理

| 能力 | 说明 | 本仓库适配器 README 约定 |
|------|------|---------------------------|
| 可用券列表 | 查询会员在某门店可用券 | `coupon_list(card_no, store_id)` |
| 券码核销 | 核销指定券码 | `coupon_use(code, store_id, cashier, amount)` |

### 2.5 金额与格式约定（README 与常见接口）

- **金额单位**：**分**（1 元 = 100）。
- **日期**：YYYY-MM-DD；日期时间：YYYY-MM-DD HH:mm:ss。
- **支付方式码**（示例）：1 现金、2 银行卡、3 店内微信、4 店内支付宝、6 线上微信、8 美团、9 大众点评（以语雀文档为准）。

## 三、从语雀文档需核对的要点

打开 [会员网关 · 语雀](https://www.yuque.com/acewillomp/mmag8t/iynakxn7fnz3pfkp) 后，建议逐项核对并补充到本仓库：

1. **基础信息**  
   - 请求 Base URL（是否与 尝在一起 奥琦玮微生活 Base URL 一致）。  
   - 认证方式：是否使用 API-AppId、API-AppKey；是否还有 AppSecret；请求头/Query 命名（如 X-App-Id、appKey）。

2. **签名规则**  
   - 参数排序、拼接方式、是否含 AppSecret、MD5/其他算法、大小写。

3. **会员相关接口**  
   - 查询会员：路径、请求参数（卡号/手机/openid）、响应字段（余额、积分、等级等）。  
   - 新增/修改会员：路径、必填项、门店 ID 是否使用 尝在一起 门店 ID（如永安店 3754547598、文化城店 2841412315）。

4. **交易/储值/优惠券**  
   - 交易预览/提交：路径、参数（门店、金额、商品、支付方式、抵扣方案）。  
   - 储值：路径、参数、响应余额单位。  
   - 优惠券：路径、核销参数与响应。

5. **错误码与报文格式**  
   - 统一响应结构（如 code/msg/data）；常见错误码及含义。

6. **限流与环境**  
   - 测试/生产环境区分；限流或频率限制说明。

## 五、本仓库现有实现与后续可做

- **packages/api-adapters/aoqiwei**：当前主要实现为**奥琦玮供应链**（openapi.acescm.cn，AppKey+AppSecret），包含库存、POS 订单上传、日结等；README 中描述的 微生活 会员/交易/储值/优惠券 方法为设计约定，需按语雀会员网关文档实现或对齐。
- **建议**：在语雀文档中确认「会员网关」的 Base URL、路径与认证方式；若与现有 `AOQIWEI_*` 不同，可使用 `CHANGZAIYIQI_AOQIWEI_APP_ID` / `CHANGZAIYIQI_AOQIWEI_APP_KEY` 及单独 Base URL 实现微生活会员网关客户端，并与 [尝在一起-奥琦玮微生活与门店配置](./尝在一起-奥琦玮微生活与门店配置.md) 中的门店 ID 一致。
