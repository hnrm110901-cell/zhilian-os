# 尝在一起 · 奥琦玮微生活接口与门店配置

**品牌**：尝在一起  
**用途**：奥琦玮微生活接口参数及尝在一起/浏览在一起门店 ID 映射，供屯象OS 对接与报表使用。

---

## 一、奥琦玮微生活 · 尝在一起接口参数

奥琦玮微生活为尝在一起提供的接口采用 **API-AppId + API-AppKey** 认证。

| 参数名 | 说明 | 配置位置 |
|--------|------|----------|
| **API-AppId** | 应用/商户标识 | 环境变量 `CHANGZAIYIQI_AOQIWEI_APP_ID` 或 奥琦玮微生活后台 |
| **API-AppKey** | 应用密钥，参与签名或请求头 | 环境变量 `CHANGZAIYIQI_AOQIWEI_APP_KEY`（**勿提交到仓库**，仅写在本地 `.env`） |

- 实际 **API-AppId**、**API-AppKey** 请在本地 `.env` 中配置，或通过配置中心/密钥管理注入。  
- 若奥琦玮微生活与现有奥琦玮供应链（`AOQIWEI_APP_KEY` / `AOQIWEI_APP_SECRET`）为同一套，可直接复用；若为独立微生活接口，则使用上表单独配置。

## 三、环境变量示例（.env，勿提交密钥）

在项目根目录或 api-gateway 的 `.env` 中可增加（值为占位，实际值勿提交到 Git）：

```bash
# 尝在一起 · 奥琦玮微生活（若与 AOQIWEI 供应链分离则使用）
CHANGZAIYIQI_AOQIWEI_APP_ID=your-api-app-id
CHANGZAIYIQI_AOQIWEI_APP_KEY=your-api-app-key
CHANGZAIYIQI_AOQIWEI_BASE_URL=https://api.aoqiwei.com

# 尝在一起门店 ID（可选，用于脚本或报表按店筛选）
CHANGZAIYIQI_STORE_YONGAN=3754547598
CHANGZAIYIQI_STORE_WENHUACHENG=2841412315
LIULANZAIYIQI_STORE_ID=4169892792
```

- 若微生活与现有 `AOQIWEI_*` 共用同一套 base_url / 认证，可只配置 `AOQIWEI_APP_KEY`、`AOQIWEI_APP_SECRET`，并在代码中按需映射 AppId/AppKey 与现有参数。

## 四、与品智收银的区分

| 系统 | 认证方式 | 用途 |
|------|----------|------|
| **品智收银** | PINZHI_TOKEN + PINZHI_BASE_URL | 尝在一起收银/经营数据（门店、每日经营、订单等），见《品智餐饮API接口参数汇总》 |
| **奥琦玮微生活** | API-AppId + API-AppKey | 尝在一起在奥琦玮侧的微生活/供应链等接口 |

两边可能同时使用，报表或拉数时需区分数据来源（品智 vs 奥琦玮微生活）。
