## 背景

徐记海鲜是智链OS的首个飞行员客户，门店规模：3-20家连锁。
核心验证目标：帮助连锁老板每年多赚30万+（成本率降低2个点）。

## 验证流程

### 1. 业务规模适配检查
- [ ] 该功能是否符合 3-20 门店链条的业务规模假设
- [ ] 数据量假设是否合理（单店日均订单 200-500 笔）
- [ ] 多租户隔离是否正确（brand_id + store_id 两级）
- [ ] 跨店对标功能是否在 2+ 门店时有意义

### 2. 企业微信集成检查
- [ ] 推送消息是否通过 WeChatService 发送
- [ ] 消息格式是否符合企微卡片消息规范
- [ ] 一键操作按钮是否有回调接口
- [ ] 推送频率是否合理（不扰民：每日≤3条关键推送）

### 3. 老板看板可见性评估
- [ ] 该功能的关键指标是否出现在 BFF 聚合响应中
- [ ] 是否有对应的前端可视化组件（ZKpi/HealthRing/ChartTrend）
- [ ] 数据刷新延迟是否≤30秒（Redis缓存 + refresh=true）
- [ ] 离线降级时是否返回缓存数据而非空白

### 4. ¥价值量化
- [ ] 功能输出是否包含明确的¥金额影响
- [ ] 是否可计算 ROI（投入时间 vs 预期节省）
- [ ] 是否可被 `case_story_generator.py` 采集为成功案例数据
