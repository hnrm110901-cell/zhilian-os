# /consult — 餐饮运营咨询诊断

当用户输入 `/consult <问题或场景>` 时，协调 Consultant + 对应角色 Agent 进行专业运营诊断。

用法：
- `/consult 食材成本率偏高怎么办` — 成本诊断
- `/consult 新店选址评估` — 战略咨询
- `/consult 翻台率提升方案` — 运营优化
- `/consult 会员复购率下降` — 私域诊断

## 执行流程

### Phase 1: 问题分类与角色调度

根据问题关键词自动匹配：

| 问题域 | 主导 Agent | 辅助 Agent | 数据需求 |
|--------|-----------|-----------|---------|
| 成本/损耗 | Consultant | Head Chef + Store Manager | 采购、库存、BOM、盘点 |
| 营收/客流 | Consultant | Floor Manager + HQ Director | 订单、客流、客单价 |
| 人力/排班 | Consultant | Store Manager | 排班、人效、工时 |
| 会员/私域 | Consultant | Floor Manager | RFM、复购、流失 |
| 多店/战略 | Consultant + HQ Director | — | 多店 KPI、财务 |
| 食安/品质 | Consultant | Head Chef | 检查记录、客诉 |

### Phase 2: 数据诊断（Consultant Agent 主导）

1. 明确问题边界（哪些门店？什么时间段？）
2. 识别相关的系统数据源：
   - 品智 POS API（营收、订单、菜品销量）
   - 库存系统（采购、盘点、损耗）
   - 会员系统（RFM、复购、流失率）
   - 排班系统（工时、人效）
3. 用行业标杆对比，量化差距（¥ 金额）

### Phase 3: 角色验证

由对应角色 Agent 从一线视角验证方案可行性：
- **Store Manager**：这个方案店长能执行吗？需要什么资源？
- **Head Chef**：后厨操作能配合吗？会增加多少工作量？
- **Floor Manager**：前厅服务会受影响吗？客户感知如何？
- **HQ Director**：多店推广可行吗？投资回报如何？

### Phase 4: 输出诊断报告

```
# 运营诊断报告

## 问题：{一句话}
## 诊断日期：YYYY-MM-DD
## 涉及门店：{门店列表}

## 诊断结论
（3句话概括核心发现）

## 关键数据
| 指标 | 当前值 | 标杆值 | 差距 | ¥影响/月 |
|------|--------|--------|------|---------|

## 根因分析
1. {根因1} — 占比 XX%
2. {根因2} — 占比 XX%
3. {根因3} — 占比 XX%

## 行动方案（按 ¥ 影响排序）

### 方案 1：{标题}（预期 ¥XX,XXX/月）
- 动作：...
- 责任人：店长/厨师长/总部
- 落地周期：X 周
- 一线可行性：✅ 已由 {角色Agent} 验证

### 方案 2：...

## 投入产出
- 总投入：¥XX,XXX
- 预期年化收益：¥XX,XXX
- 回收周期：X 个月

## 跟踪指标
监控以下指标验证效果（建议 4 周后复查）：
1. {指标} — 目标值：XX
2. {指标} — 目标值：XX
```
