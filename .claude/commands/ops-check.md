# /ops-check — 门店运营健康度巡检

当用户输入 `/ops-check` 时，从四个角色视角对系统功能做全方位运营适配性检查。

用法：
- `/ops-check` — 全角色巡检
- `/ops-check sm` — 只检查店长视角
- `/ops-check chef` — 只检查厨师长视角
- `/ops-check floor` — 只检查楼面经理视角
- `/ops-check hq` — 只检查总部视角
- `/ops-check <feature_name>` — 针对特定功能做全角色评估

## 执行流程

### Phase 1: 确定检查范围

1. 识别需要检查的功能模块或页面
2. 读取对应的前端页面代码（`apps/web/src/pages/{role}/`）
3. 读取对应的 BFF 端点（`/api/v1/bff/{role}/{store_id}`）
4. 读取对应的后端服务

### Phase 2: 四角色并行评审

同时从四个角色视角评估（使用对应 Agent 规范）：

**Store Manager Agent** — 店长评审：
- 早班会/午市/采购/晚市/日结 五个场景覆盖度
- 3 秒看懂？有行动建议？有 ¥ 量化？
- 手机操作流畅度

**Head Chef Agent** — 厨师长评审：
- 备货/出品/损耗/食安/带教 五个战场覆盖度
- 语音可操作？大字清晰？3 步完成？
- 预警及时性

**Floor Manager Agent** — 楼面经理评审：
- 排队/桌台/预订/服务/客诉/VIP 六个模块覆盖度
- 平板操作？高峰期不过载？
- 翻台帮助度

**HQ Director Agent** — 总部评审：
- 全景/财务/对标/决策 四个模块覆盖度
- 多店一目了然？异常秒发现？
- 能给投资人看？
