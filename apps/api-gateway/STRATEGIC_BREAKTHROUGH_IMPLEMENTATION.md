## 一、战略转型概览

### 1.1 核心转变

**From**: 技术驱动的SaaS系统
**To**: 商业结果驱动的RaaS服务

**From**: 卖软件功能
**To**: 卖省钱和赚钱的结果

**From**: 订阅制收费（¥999/月）
**To**: 效果付费制（省下成本的20%，增加营收的15%）

### 1.2 新定位

**智链OS = AI数字总经理**

- 不是软件，是数字总经理
- 不卖功能，卖结果
- 不靠技术，靠网络效应
- 不做巨头，做生态

## 二、实施内容

### 2.1 核心模块开发

#### Module 1: RaaS定价服务

**文件**: `src/services/raas_pricing_service.py`
**代码行数**: 280行

**核心功能**:
1. **定价层级管理**
   - FREE_TRIAL: 基础版（免费试用3个月）
   - COST_SAVING: 效果版（按省下的成本分成20%）
   - REVENUE_GROWTH: 增长版（按增加的营收分成15%）
   - MODEL_MARKETPLACE: 模型版（一次性购买）

2. **基线指标计算**
   - 成本基线：食材损耗率、人工成本、能源成本、库存周转率
   - 营收基线：日营业额、客流量、客单价、复购率
   - 基线时间范围：试用期前3个月

3. **效果指标计算**
   - 成本节省：食材损耗节省、人工成本节省、能源成本节省、库存成本节省
   - 营收增长：客流增加、客单价提升、复购率提升
   - 计费金额：成本节省分成20%，营收增长分成15%

4. **月度账单生成**
   - 对比基线和当前期间数据
   - 计算实际产生的效果
   - 生成透明的账单

**商业价值**:
- 降低客户付费门槛（3个月免费试用）
- 按效果付费，客户更愿意尝试
- 透明的计费方式，建立信任

---

#### Module 2: 模型交易市场服务

**文件**: `src/services/model_marketplace_service.py`
**代码行数**: 260行

**核心功能**:
1. **模型分级**
   - Level 1 (Basic): 基础服务（免费） - 使用自己门店数据训练的模型
   - Level 2 (Industry): 行业模型（¥9,999/年） - 全国1000家门店联邦训练的通用模型
   - Level 3 (Custom): 定制模型（¥29,999/年） - 针对特定品类的专属模型

2. **模型类型**
   - 排班模型 (Scheduling)
   - 库存预测模型 (Inventory)
   - 动态定价模型 (Pricing)
   - BOM损耗控制模型 (BOM)
   - 客户流失预测模型 (Customer Churn)
   - 需求预测模型 (Demand Forecast)

3. **行业分类**
   - 火锅 (Hotpot)
   - 烧烤 (BBQ)
   - 快餐 (Fast Food)
   - 中餐 (Chinese Restaurant)
   - 海鲜 (Seafood)
   - 西餐 (Western Food)

4. **数据贡献分成**
   - 门店贡献数据参与联邦学习
   - 获得模型销售收益的30%分成
   - 数据质量越高，分成越多

5. **网络效应计算**
   - 总接入门店数
   - 总模型数量
   - 总训练数据点
   - 平均模型准确率
   - 网络价值（梅特卡夫定律：n^2 * 单位价值）

**商业价值**:
- 构建护城河：网络效应，用的店越多，壁垒越高
- 新收入来源：模型销售收入
- 激励数据贡献：门店愿意分享数据
- 快速复制：新店立即获得成熟经验

#### Module 3: Human-in-the-Loop服务（增强版）

**文件**: `src/services/human_in_the_loop_service.py`
**代码行数**: 380行

**核心功能**:
1. **风险等级分类**
   - Level 1 (Low): 自动执行（查询、分析、报表、通知）
   - Level 2 (Medium): 自动执行+事后审计（自动排班、小额采购、优惠券发放）
   - Level 3 (High): 人工审批（大额采购>5000元、人员调动、价格调整、供应商变更）
   - Level 4 (Critical): 禁止AI操作（资金打款、数据删除、权限变更、合同签署）

2. **信任阶段管理**
   - 观察期（1-3个月）：AI只提建议，不执行
   - 辅助期（3-6个月）：AI执行低风险操作，高风险需审批
   - 自主期（6个月+）：AI自主执行大部分操作，仅极高风险需审批

3. **审批流程**
   - AI决策提交
   - 风险等级判断
   - 自动执行或人工审批
   - 企业微信推送通知
   - 店长审批
   - 执行操作

4. **审批统计**
   - 总决策数
   - 自动执行数
   - 审批请求数
   - 审批通过率
   - 平均审批时间
   - 信任评分

**商业价值**:
- 建立信任：逐步放权，避免"大模型发疯"
- 降低风险：高危操作必须人工审批
- 提升可靠性：AI决策可追溯可解释
- 符合监管：满足企业合规要求

## 七、安全防线

### 7.1 Human-in-the-Loop架构

**核心原则**: "建立信任需要三年，摧毁信任只需大模型发疯一次"

**架构**:
```
AI决策层（System 1）
    ↓
规则校验层（System 2）
    ↓
触发红线？
 /    \
是      否
↓       ↓
人工审批  自动执行
```

### 7.2 高危操作分级

- **Level 1**: 自动执行（低风险）
- **Level 2**: 自动执行+事后审计（中风险）
- **Level 3**: 人工审批（高风险）
- **Level 4**: 禁止AI操作（极高风险）

### 7.3 信任建立机制

**阶段1: 观察期**（1-3个月）
- AI只提建议，不执行
- 人工对比AI建议和实际决策
- 建立信任基础

**阶段2: 辅助期**（3-6个月）
- AI执行低风险操作
- 高风险操作需审批
- 逐步放权

**阶段3: 自主期**（6个月+）
- AI自主执行大部分操作
- 仅极高风险需审批
- 完全信任

## 九、技术实现亮点

### 9.1 RaaS定价算法

**基线计算**:
- 收集试用期前3个月的运营数据
- 计算平均成本和营收指标
- 作为效果对比的基准

**效果计算**:
- 对比基线和当前期间数据
- 计算实际产生的成本节省和营收增长
- 按比例计算服务费

**透明度**:
- 所有计算过程可追溯
- 客户可以查看详细的效果指标
- 建立信任

**数据贡献激励**:
- 门店贡献数据获得分成

- 激励高质量数据贡献

### 9.3 Human-in-the-Loop

**风险分类算法**:
