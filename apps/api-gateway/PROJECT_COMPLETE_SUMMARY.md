## 一、项目概览

### 1.1 项目定位

屯象OS是一个**AI Native的餐饮SaaS 2.0系统**，定位为传统SaaS的"AI副驾驶"，悬浮在美团/天财商龙等底层交易系统之上，提供智能决策和运营优化能力。

### 1.2 核心价值

- **AI Native**: 真正的智能体化架构，7个专业Agent协同工作
- **无屏幕交互**: 语音交互革命，解放双手
- **联邦学习**: 跨门店知识共享，数据不出域
- **SOP知识库**: 沉淀顶级门店运营经验，构建护城河

## 五、核心模块详解

### 5.1 多租户数据隔离

**实现方式**: 4层防护
- 中间件层：自动注入tenant_id
- 应用层：ContextVar线程安全
- ORM层：SQLAlchemy会话级过滤
- 数据库层：PostgreSQL RLS

**文件**: `src/core/tenant_context.py`, `src/core/tenant_filter.py`

### 5.2 神经/符号双规机制

**核心理念**: System 1 (Neural) + System 2 (Symbolic)
- 12条硬性规则（财务/运营/安全/合规/业务）
- 触发红线立即降级为人类审批
- AI决策可靠性95%+

**文件**: `src/core/neural_symbolic_guardrails.py`

### 5.3 多模态优雅降级

**降级链路**: 语音→手表→POS→KDS→企微
- 环境噪音检测（>85dB自动切换）
- ASR失败2次立即降级
- 关键指令多通道并发

**文件**: `src/services/multimodal_fallback_service.py`

### 5.4 联邦学习

**应用场景**: 跨门店学习食材损耗规律
- 数据不出域，只共享模型参数
- FedAvg聚合算法
- 差分隐私保护

**文件**: `src/services/federated_learning_service.py`, `src/services/federated_bom_service.py`

### 5.5 SOP知识库

**核心价值**: 沉淀顶级门店运营知识
- 4类预置SOP（客诉/爆单/安全/设备）
- RAG检索增强生成
- 打造"虚拟店长"产品

**文件**: `src/services/sop_knowledge_base_service.py`

### 5.6 嵌入模型

**功能**: 语义理解和智能推荐
- Word2Vec Skip-gram训练
- 相似度计算（余弦、欧氏）
- 智能推荐（相似菜品、订单推荐）

**文件**: `src/services/embedding_model_service.py`

### 5.7 安全防护

**向量数据库隔离**:
- Collection级物理隔离
- Payload Filter强制校验
- 防止跨租户数据泄露

**KMS密钥管理**:
- 密钥加密存储（Fernet + PBKDF2）
- 运行时动态解密
- 密钥轮换机制（90天）

**Prompt注入防护**:
- 40+危险模式检测
- 输入清洗网关（3级清洗）
- 数据与指令严格分离

**文件**: `src/core/vector_db_security.py`, `src/core/kms_service.py`, `src/core/prompt_injection_guard.py`

- Python 3.9+
- PostgreSQL 12+
- Redis 6+
- Qdrant向量数据库

### A.2 安装部署

```bash
# 1. 克隆仓库
git clone https://github.com/hnrm110901-cell/zhilian-os.git
cd zhilian-os/apps/api-gateway

# 2. 安装依赖
pip install -r requirements.txt

# 3. 数据库迁移
alembic upgrade head

# 4. 启动服务
uvicorn src.main:app --host 0.0.0.0 --port 8000
```
